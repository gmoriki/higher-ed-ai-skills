## 現在の方針（初期フェーズ）

本リポジトリは公開後6か月間を「初期フェーズ」と位置づけ、以下の方針で運営します。

| 貢献の種類 | 受け入れ状況 |
|-----------|-------------|
| Issue（質問・要望・バグ報告） | 歓迎 |
| Pull Request（誤字修正・リンク切れ修正・事例追加） | 歓迎 |
| Pull Request（新スキルの追加） | 当面は受け入れ不可（後述） |
| GitHub Discussions | 歓迎 |

### 新スキルの追加について

初期フェーズでは、新しいスキルの追加はリポジトリ管理者（森木）のみが行います。これは品質管理のためです。

skill.md は大学業務の判断基準を含むため、不正確な内容が含まれると実務に影響を及ぼす可能性があります。スキルの品質基準が確立し、レビュー体制が整うまでは、管理者が責任を持って作成・更新します。

「こういうスキルがほしい」という要望は、Issue または Discussions で歓迎します。

## Issue の書き方

Issue は以下の3種類を想定しています。内容に応じて適切なテンプレートを使用してください。

### バグ報告（スキルの誤り・リンク切れなど）

```markdown
## 報告対象

- スキル名 / ファイルパス:
- バージョン:

## 問題の内容

（何が間違っているか、期待される内容は何かを具体的に記述してください）

## 再現手順（該当する場合）

1.
2.
3.

## 補足情報

（スクリーンショット、参考URL、関連する法令・通知など）
```

### 機能要望（新スキルの提案・既存スキルの改善）

```markdown
## 要望の種類

- [ ] 新しいスキルの提案
- [ ] 既存スキルの改善
- [ ] その他

## 概要

（どのような機能・スキルがほしいかを簡潔に記述してください）

## 背景・動機

（なぜこの機能・スキルが必要か。どのような業務場面で使いたいか）

## 想定される利用者

（教員 / 職員 / その他）

## 参考情報

（関連する資料、他大学の事例、参考URLなど）
```

### 質問

```markdown
## 質問の内容

（具体的な質問を記述してください）

## 試したこと

（自分で調べたこと、試したことがあれば記述してください）

## 関連するスキル・ドキュメント

（質問に関連するファイルがあればパスを記述してください）
```

## Pull Request の書き方

### 受け入れ可能な Pull Request（初期フェーズ）

- 誤字・脱字の修正
- リンク切れの修正
- 事例の追加（`case-studies/` 配下）
- references の情報更新（AIサービスのポリシー変更など）
- ドキュメントの改善（わかりにくい表現の修正など）

### PR テンプレート

```markdown
## 変更内容

（何を変更したかを簡潔に記述してください）

## 変更理由

（なぜこの変更が必要かを記述してください）

## 確認事項

- [ ] 変更内容は事実に基づいている
- [ ] リンクが正しく機能する
- [ ] 既存の内容との整合性を確認した
```

### PR を出す前の確認事項

1. **ブランチ**: `main` ブランチから新しいブランチを作成し、そこで作業してください。
2. **コミットメッセージ**: 変更内容がわかるメッセージを付けてください。
3. **1つのPRに1つの変更**: 複数の修正がある場合は、できるだけPRを分けてください。
4. **スキル本体の変更**: スキルの本体（SKILL.md）を変更する場合は、事前に Issue で相談してください。

## GitHub Discussions

大学におけるAI活用全般について、自由に議論できる場として GitHub Discussions を開放しています。

想定されるトピック:

- 大学業務でのAI活用事例の共有
- スキルの使い方に関する相談
- AI関連の法令・ガイドラインに関する情報交換
- ランタイム（ChatGPT、Copilot、Claude など）の比較や活用Tips
- 大学のAIガイドライン策定に関する議論

Issue にするほどではない雑談や意見交換も歓迎します。

## 行動規範（Code of Conduct）

本プロジェクトでは、すべての参加者に対して敬意ある行動を求めます。

- 建設的なフィードバックを心がけてください
- 異なる意見や経験を尊重してください
- 個人への攻撃やハラスメントは許容しません
- 大学名や個人名を挙げての批判は避けてください

問題のある行動を目撃した場合は、リポジトリ管理者に連絡してください。

## 中長期の方針

公開後6か月を目処に、以下の変更を予定しています。

### Contributor 制度の導入

- 継続的に貢献いただいた方を Contributor として認定する仕組みを導入します
- Contributor は、新スキルの提案・レビューに参加できるようになります

### 外部からの新スキル提案の受け入れ

- レビュー体制が整った段階で、外部からの新スキル追加 PR も受け入れを開始します
- 新スキルの品質基準とレビュープロセスは、その時点で改めて公開します

### レビュー体制

- 新スキルのレビューには、大学業務の実務経験者が参加する体制を目指します
- 法令や規程に関する記述は、必要に応じて専門家の確認を経ます
